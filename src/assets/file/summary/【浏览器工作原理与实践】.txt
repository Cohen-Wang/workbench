【浏览器】
软件
标准：浏览器就是基于一套套标准实现。
	页面标准
		w3c
		whatwc
	语言标准
		ECMAScript
		WebAssembly
	网络标准
		rfc-xxx
	废弃的标准
		WebSQL：web使用sql语言
		FileSystem：文件形式读取数据
		Application Cache：页面内容缓存到本地。（后来改造为PWA）

浏览器背后是标准，标准的背后是共识。

前端发展优势
	开放式平台
	渲染效率逐渐接近原生
	发展快
		技术大爆炸
		支持业务类型越来越多
		每种技术又在迭代更新
		
浏览器的知识体系
	1.浏览器的架构设计
		早期：单进程浏览器：页面渲染，页面展现，js环境，插件，都运行在页面线程中。所以一个页面的崩溃，卡顿，都会影响到其他页面。
		现在：多进程架构，每个页面都运行在单独的进程里面。此时，页面已经可以看成是一个单独的web应用了。
		
	2.javascript引擎工作原理
		站在浏览器引擎的视角来分析代码的执行流程。
	
	3.页面的工作原理
		前端工程师所做的一切最终都是为了页面的展示。
		事件循环系统：是页面的心脏。单线程：一次只能处理一个事件。
		页面的核心流程：数据从接收到页面显示的核心流程。则对于页面性能问题将是小菜一碟。
		
	4.浏览器网络
		http cookie cache tcp dns 管线化等概念。
		https http2 http3 websocket等概念。
		
	5.浏览器安全
		浏览器对网络不信任。
		1.网络传输过程中，数据可能被串改或窃取。
		2.操作系统层面，恶意的web应用可以通过浏览器的漏洞来攻击操作系统。
		3.浏览器层面，恶意的程序获取浏览器页面的一些隐私信息。
			沙箱，https，xss，csrf，同源策略等。
